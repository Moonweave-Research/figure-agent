# Issue 100AT - Run Journal New Evidence Snapshot Gap

Status: implemented

Type: continuation UX, freshness hardening, run journal contract

## Problem

Issue 100AR made recorded fig_run journals hash-check files that existed when
the journal was written. Follow-up review found the complementary gap: a new
evidence file can appear after the journal, but if its mtime is older than the
journal, the legacy mtime fallback will not mark the journal stale and the hash
snapshot has no entry to compare.

That can happen when evidence is copied from another checkout, restored from an
archive, or generated by a tool that preserves timestamps.

## Decision

For journals that include `evidence_snapshot`, compare the current fixture
evidence source set with the snapshot item paths:

- changed or missing snapshotted files still mark the journal stale;
- newly present evidence files that were absent from the snapshot also mark the
  journal stale, regardless of mtime;
- old journals without a snapshot keep the legacy mtime-only fallback for
  backward compatibility.

The run journal remains non-authoritative and does not allow replay.
