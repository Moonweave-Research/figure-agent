# Issue 12A: High-Zoom Visual Audit Crop Pack

**Date:** 2026-05-20 KST
**Status:** implemented on main
**Type:** AFK implementation slice
**Parent:** `2026-05-20-issue-12-critical-visual-audit-gaps.md`

## What To Build

Add a deterministic high-zoom visual audit pack to `/fig_critique`. The pack
creates 2x2 attention crops from the current compiled render and from any panel
crops that `critique_brief.py` already generates. The generated brief must list
these crop paths and require the host LLM to inspect them for micro visual
defects before writing `critique.md`.

These crops are not semantic sub-regions. They must not be treated as
`subregion_iteration_log.md` targets, patch units, or cross-panel active
sub-regions. They are only evidence images for the host critique.

This slice does not make crop usage mechanically enforceable. It makes the crop
evidence available and command-facing. Schema enforcement belongs to Issue 12B.

This is not a new vision API and not a new patching tool. It only improves the
evidence that the existing host-LLM critique sees.

## Acceptance Criteria

- [x] `scripts/critique_zoom_crops.py` creates deterministic 2x2 crop images
  under `examples/<name>/build/audit_crops/`.
- [x] Full-render crops are created for every fixture with a fresh build PNG.
- [x] Panel-level crops are created for every panel crop generated by the
  existing per-panel reference workflow.
- [x] The brief includes `## High-Zoom Visual Audit Crops`.
- [x] The brief lists every crop path relative to `examples/<name>/`.
- [x] The brief instructs the host LLM to inspect the crops for:
  `line_crosses_label`, `wire_crosses_label`, `arrow_tip_fused`,
  `label_target_detached`, `floating_semantic_cue`, and
  `drawing_order_suspect`.
- [x] Existing panel-reference warning behavior is preserved.
- [x] Existing fixtures with no panel references still get full-render crops.
- [x] No source `.tex`, `critique.md`, adjudication, export, accepted, golden,
  or final-artifact file is mutated.
- [x] Focused tests, full tests, ruff, diff check, and plugin validation pass.
