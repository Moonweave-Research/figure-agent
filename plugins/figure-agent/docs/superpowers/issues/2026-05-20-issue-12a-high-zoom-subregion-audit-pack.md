# Issue 12A: High-Zoom Subregion Audit Pack

**Date:** 2026-05-20 KST
**Status:** open
**Type:** AFK implementation slice
**Parent:** `2026-05-20-issue-12-critical-visual-audit-gaps.md`

## What To Build

Add a deterministic high-zoom audit pack to `/fig_critique`. The pack creates
zoom crops from the current compiled render and from any panel crops that
`critique_brief.py` already generates. The generated brief must list these crop
paths and require the host LLM to inspect them for micro visual defects before
writing `critique.md`.

This is not a new vision API and not a new patching tool. It only improves the
evidence that the existing host-LLM critique sees.

## Acceptance Criteria

- [ ] `scripts/critique_zoom_crops.py` creates deterministic 2x2 crop images
  under `examples/<name>/build/audit_crops/`.
- [ ] Full-render crops are created for every fixture with a fresh build PNG.
- [ ] Panel-level crops are created for every panel crop generated by the
  existing per-panel reference workflow.
- [ ] The brief includes `## High-Zoom Visual Audit Crops`.
- [ ] The brief lists every crop path relative to `examples/<name>/`.
- [ ] The brief instructs the host LLM to inspect the crops for:
  `line_crosses_label`, `wire_crosses_label`, `arrow_tip_fused`,
  `label_target_detached`, `floating_semantic_cue`, and
  `drawing_order_suspect`.
- [ ] Existing panel-reference warning behavior is preserved.
- [ ] Existing fixtures with no panel references still get full-render crops.
- [ ] No source `.tex`, `critique.md`, adjudication, export, accepted, golden,
  or final-artifact file is mutated.
- [ ] Focused tests, full tests, ruff, diff check, and plugin validation pass.
