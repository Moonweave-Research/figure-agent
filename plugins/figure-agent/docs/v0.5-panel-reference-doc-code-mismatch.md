# v0.5 Per-Panel Reference — Doc ↔ Code Mismatch

**Status:** ISSUE (pre-design). No fix committed.
**Filed:** 2026-05-16
**Severity:** MEDIUM — silent feature gap. Compile and critique both succeed; user gets less per-panel coverage than the doc promises.
**Surfaced by:** 2026-05-16 dogfood attempt to "wake up" Panel B–G via bbox-only registration in `examples/fig1_overview_v2_pair_001_vault/spec.yaml`.

---

## 1. Problem

The v0.5 per-panel reference workflow doc claims a fallback that the shipped code does not implement. Panels in `spec.yaml.panels[]` that declare `bbox_pdf_cm` **without** `reference_image` are **silently skipped** by `critique_brief.py`, even though the doc says they should fall back to the figure-level reference.

## 2. Evidence

**Doc** — `plugins/figure-agent/docs/architecture-v0.5-per-panel-reference-workflow.md`:

- Line 53 (§3.1 Compatibility):
  > If `panels[i].reference_image` is present, v0.5 critique uses it for that panel.
  > If absent, **falls back to the existing figure-level `reference_image`** for that panel (current behavior, no regression).

- Line 109 (§ Acceptance criteria):
  > Absence of either field **falls back to figure-level reference comparison** (no regression on existing fixtures).

**Code** — `plugins/figure-agent/scripts/critique_brief.py` lines 195–217:

```python
for index, panel in enumerate(spec.get("panels", [])):
    ref_path = _panel_reference_path(example_dir, panel)
    if ref_path is None:
        continue                       # <- panels without reference_image
                                       #    are skipped, NOT folded into
                                       #    figure-level fallback
    ...
```

`_panel_reference_path` returns `None` whenever `panel.get("reference_image")` is missing or empty. The `continue` then bypasses crop generation and the per-panel context block entirely. No warning, no fallback path.

## 3. Impact

Concrete dogfood case (2026-05-16):

- `examples/fig1_overview_v2_pair_001_vault/spec.yaml` was extended to register all 7 panels with bbox_pdf_cm (commit `cb33377`).
- Panel A has a dedicated `reference_image`. Panels B–G omit it, expecting the documented fallback.
- `critique_brief.py` output: `## Per-panel reference contexts` lists **only Panel A**. Panels B–G are absent.
- Result: 7-panel registration produces 1-panel critique coverage. Author who trusts the doc sees no warning and no panel-specific feedback for B–G.

This is the root cause of why "registering panels in spec.yaml" did not unlock panel-grounded critique for B–G in the active dogfood arm.

## 4. Options

Three resolutions, listed without recommendation; the choice is a deliberate v0.5 product decision.

1. **Update doc to match code (panel needs both fields)** — Smallest change, no migration. v0.5 becomes "per-panel critique requires both `reference_image` AND `bbox_pdf_cm`; absence of either disables that panel". Cost: removes the no-regression guarantee in current spec.

2. **Update code to match doc (fall back to figure-level reference)** — Honors the no-regression promise. Implementation: when `panel.reference_image` is missing but `bbox_pdf_cm` is present, generate the crop and use the figure-level reference for comparison. Concern: comparing a small panel crop against a full-figure overview reference is degenerate — the visual scales differ. Per-panel finding may be noise unless the crop is paired with a panel-scaled reference crop.

3. **Add WARN on silent skip (no behavior change, observability fix only)** — Code keeps skipping; brief emits `WARN: Panel <id> has bbox_pdf_cm but no reference_image; skipping per-panel comparison` so the author knows. Cheapest. Does not deliver the documented fallback, only stops it being silent.

Option 1 is honest. Option 2 honors the doc but may produce useless findings. Option 3 is a partial fix that mostly addresses "silent" not "missing".

## 5. Non-goals

- ❌ Auto-cropping the figure-level reference into per-panel pieces. Would mask the same scale issue as Option 2 with extra machinery.
- ❌ LLM-generated panel reference images. Re-enters falsified family (`project_v0_3_figure_quality_spec_rejected.md`).

## 6. Open questions

- Which of (1)–(3) was the original v0.5 intent? Architecture doc clearly promises fallback; code clearly skips. One author meant the other.
- If Option 2: should the fallback use a cropped figure-level reference (auto-cropped per panel bbox), or the full figure-level reference, or refuse with WARN?
- Is silent skip ever the right behavior? Option 3's WARN is cheap and benefits even if (1) or (2) lands later.

## 7. Decision gate

No design committed. Resolution requires picking among (1)–(3). Recommend pairing whichever path is chosen with (3) regardless, so the silent-skip surface is closed.

Until resolved, **author guidance**: every panel in `spec.yaml.panels[]` that should receive panel-grounded critique must have **both** `reference_image` and `bbox_pdf_cm`. Treat the doc's fallback claim as a non-promise.
