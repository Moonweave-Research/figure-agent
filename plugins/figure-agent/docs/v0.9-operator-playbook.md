# v0.9 Operator Playbook

**Status:** release-candidate operating model
**Audience:** human or agent operator running figure-agent on real fixtures

This playbook is the short path through the v0.9 workflow. It does not replace
the command docs; it tells you which command to use first and where automation
must stop.

## Core Rule

Use the driver surfaces as traffic controllers:

- Single fixture: start with `/fig_status` or `/fig_drive --dry-run`.
- Repeated one-fixture improvement: use `/fig_improve`.
- Multi-fixture: start with `/fig_queue`.
- Mechanical execution: use `/fig_run` or `/fig_queue_run` only after inspecting
  the dry-run plan.

Never infer permission from a stale `safe_command` in an old log. Rerun live
status or driver state first.

## Single Fixture

From the plugin root:

```bash
fig-agent drive <fixture> --mode review --goal "<goal>" --dry-run
```

Then follow the action:

| Action shape | What to do |
|---|---|
| `run_compile` | run `/fig_compile` or the printed compile command |
| `run_critique` | run host `/fig_critique <fixture>`; do not fake critique.md |
| `run_adjudicate` | scaffold only if adjudication is missing |
| `run_fig_loop` | record verify-only loop checkpoint |
| `run_export` | export only when the driver allows draft generated export |
| `human_gate_stop` | record the human decision before automation continues |
| `release_blocked` | perform explicit release/golden/accepted review |
| `patch_handoff_stop` | patch exactly one approved target outside hidden automation |
| `svg_polish_handoff` | complete explicit SVG polish handoff outside hidden automation |

For safe mechanical work:

```bash
fig-agent run <fixture> --mode review --goal "<goal>"
fig-agent run <fixture> --mode review --goal "<goal>" --execute
```

Default is plan-only. `--execute` still stops at host critique, human review,
existing adjudication repair, source patch, SVG polish, accepted, tracked
golden, force golden, and release boundaries.

## Loop-Centered Improvement

When the user says "use figure-agent to improve this", "loop 10 times", or
"keep reviewing until no major issue remains", prefer:

```bash
fig-agent improve <fixture> --goal "<goal>"
fig-agent improve <fixture> --goal "<goal>" --execute --max-loops 10
```

`/fig_improve` wraps `/fig_run` and emits a compact
`figure-agent.improve.v1` summary with the final required actor and the next
operator instruction. It may continue internally only after safe mechanical
steps hit the per-cycle step cap. It stops at host critique, human review,
patch handoff, SVG polish, release/golden handling, and optional-improvement
boundaries.

After the required actor acts, rerun `/fig_improve`. Do not treat
`--max-loops` as permission to author critiques, patch source, force golden, or
cross human/release gates inside one process.

## Multi-Fixture Queue

Start with the full queue:

```bash
fig-agent queue --mode review --goal "<goal>"
```

Use filters to route work:

```bash
fig-agent queue --mode review --goal "<goal>" --actor host_llm
fig-agent queue --mode review --goal "<goal>" --actor workflow_agent --command-plan --json
fig-agent queue-run --mode review --goal "<goal>" --actor workflow_agent
```

Only after reviewing the plan:

```bash
fig-agent queue-run --mode review --goal "<goal>" --actor workflow_agent --execute
```

Queue rows are grouped by required actor:

| Actor | Meaning |
|---|---|
| `workflow_agent` | deterministic shell work may be executable if no stop boundary exists |
| `host_llm` | host-vision critique must be refreshed |
| `human` | human decision or acceptance judgment is required |
| `release_operator` | release/golden/publication action needs explicit approval |
| `svg_editor` | SVG polish handoff is outside queue automation |

The command plan is the authority for batch execution. A row with
`stop_boundary` is blocked even if the driver row contains a `safe_command`.

## Host Critique Refresh

When a row says `host_llm` / `run_critique`:

1. Run `/fig_critique <fixture>` in the host vision environment.
2. Run critique lint.
3. Sync or scaffold `critique_adjudication.yaml`.
4. Rerun `/fig_queue` or `/fig_drive`.

Allowed scope is normally:

- `examples/<fixture>/critique.md`
- `examples/<fixture>/critique_adjudication.yaml`
- audit evidence read during critique

Do not edit `.tex`, accepted state, golden exports, or publication state during
a critique refresh.

## Closeout After Patch Or Export

Use closeout when a fixture looks mechanically done but still has stale export,
loop rerun, or boundary-sync questions:

```bash
fig-agent closeout <fixture> --json
```

If closeout exits 1, read the JSON anyway. It names the next blocking step.
After acting, rerun live queue/driver state. Closeout is read-only; it is not a
release approval tool.

## Release And Golden Gate

Release mode is intentionally conservative:

```bash
fig-agent queue --mode release --goal "<goal>" --json
```

Expected release blockers include:

- `accepted_or_final_ready_required`
- `force_golden_required`
- `human_gate_required`
- `host_llm_critique_required`

Do not run `--force-golden`, mutate `accepted`, or update publication
provenance as a side effect of `/fig_run` or `/fig_queue_run`. Those are
explicit release-operator decisions.

## What v0.9 Still Does Not Do

- It does not automatically produce top-tier art direction.
- It does not decide Nature/Science acceptance.
- It does not hide source patches behind runner execution.
- It does not auto-author host critiques.
- It does not auto-polish SVGs.
- It does not mutate accepted/golden/publication state.
- It does not call external provider APIs.

## Minimal Daily Loop

For one fixture:

```bash
fig-agent improve <fixture> --goal "<goal>" --execute --max-loops 10
```

For many fixtures:

```bash
fig-agent queue --mode review --goal "<goal>"
fig-agent queue --mode review --goal "<goal>" --actor host_llm
fig-agent queue-run --mode review --goal "<goal>" --actor workflow_agent
fig-agent queue-run --mode review --goal "<goal>" --actor workflow_agent --execute
```

Always resolve host/human/release/SVG rows explicitly, then rerun the queue.
