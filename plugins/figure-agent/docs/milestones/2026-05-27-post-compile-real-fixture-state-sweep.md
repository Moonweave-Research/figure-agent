# Post-Compile Real Fixture State Sweep

**Date:** 2026-05-27 KST
**Status:** implemented; two audit-evidence UX defects found and fixed

## Scope

This sweep checked the real fixture state matrix after compile artifacts exist.
It extends the earlier dry-run-only sweep by executing compile for every
published real fixture, then reading `/fig_drive` outputs across authoring,
review, polish, and release modes.

No fixture source, critique, export, accepted/golden state, SVG-polish state, or
publication provenance file was intentionally mutated. Generated build artifacts
were created for verification only and remain ignored. `smoke_trap_demo` was
exported after the sweep because the status fixture test expects exports when a
local ignored smoke build PDF exists; those generated exports also remain
ignored.

## Defects Found And Fixed

### 1. Not-applicable audit evidence suggested critique

Before fix, fixtures without `critique.md` returned:

```text
audit_evidence.evaluation_state: not_applicable
audit_evidence.next_action: /fig_critique <name>
```

That was misleading for fixtures where `critique_state=NOT_REQUIRED`.

After fix:

```text
audit_evidence.evaluation_state: not_applicable
audit_evidence.next_action: ""
audit_evidence.reason: critique.md is absent; audit evidence is not applicable
```

### 2. Audit-crop manifest problems suggested compile

Before fix, missing/malformed/stale `build/audit_crops/manifest.json` suggested
`/fig_compile`, even though the audit-crop manifest is generated by the
critique brief/crop path.

After fix, those states suggest:

```text
/fig_critique <name>
```

For the fig1 vault post-compile state, the resulting summary is:

```text
evaluation_state: missing_input
blocking_items: build/audit_crops/manifest.json
next_action: /fig_critique fig1_overview_v2_pair_001_vault
label_path.missing_refs: LP001, LP002
```

## Review

1. **Source containment:** clean. No `.tex`, critique, export, accepted/golden,
   or provenance files were edited.
2. **Generated artifacts:** clean. Compile outputs are ignored and were not
   staged.
3. **State-machine fit:** clean. Driver still respects render before critique,
   critique before export/release, and no hidden polish path.
4. **UX correctness:** fixed. Audit evidence no longer points at commands that
   cannot produce the missing artifact.
5. **Backward compatibility:** clean. Existing legacy critique handling remains
   `legacy` and still routes reference-grounded fixtures to `/fig_critique`.
6. **Test coverage:** fixed. The two misleading next-action cases are now
   covered in `tests/test_audit_evidence_summary.py`.
